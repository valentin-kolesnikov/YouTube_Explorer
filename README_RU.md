## <p align="center"><a href="README.md">English</a> | Русский</p>

**YouTube Explorer** — это консольный инструмент на Python для программного исследования контента YouTube с использованием **YouTube Data API v3** (с поддержкой OAuth 2.0), **Return YouTube Dislike API** и **YouTube Transcript API**.

> [!CAUTION]
> Оригинальный репозиторий распространяется только на `github.com`  
> Клоны, распространяющиеся от моего имени на других сайтах и `GitHub` репозиториях, не являются моей работой

Он предназначен для:

- Работы в рамках лимитов квоты YouTube API  
- Получения и фильтрации комментариев  
- Поиска видео и получения плейлистов  
- Извлечения метаданных  
- Сбора субтитров

# ❗ Требования  

Для использования YouTube Explorer вам потребуется:

1. Желание использовать **YouTube Explorer**  
2. Действующий ключ YouTube Data API v3, полученный через Google Cloud Console  
3. Python 3.10 или новее (при запуске из репозитория)  
4. Стабильное интернет-соединение  

Рекомендуется базовое знание работы с командной строкой.

# 🌐How to download?

## Releases

1. `Releases` → `YouTubeExplorer.zip`
2. Извлеките `.zip` папку
3. `bin` → откройте `YouTubeExplorer.exe`

Бинарник работает на Windows, не на Linux без `Wine` (слой совместимости, способный запускать приложения для Windows)

## Git

- `git clone https://github.com/valentin-kolesnikov/YouTube_Explorer`

## Linux

- `curl -L -o YouTubeExplorer.zip https://github.com/valentin-kolesnikov/YouTube_Explorer/releases/download/v0.9.0/YouTubeExplorer.zip`

Вы устанавливаете репозиторий с кодом. Вам понадобится скачать `Python 3.10+`. Это нужно для пользователей Linux, к сожалению.

# ❓ Как получить ключ YouTube Data API v3?

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com)  
2. Зарегистрируйтесь или войдите в аккаунт Google  
3. Нажмите `Create or select a project` → `New project`  
4. Укажите имя проекта (Google может задать его автоматически). Если у вас нет организации — не изменяйте поле `Location`. Нажмите `Create`  
5. Нажмите `Select project`, выберите проект. В поиске введите **YouTube Data API v3** → `Enable`  
6. В разделе API откройте вкладку `Credentials`  
7. Нажмите `Create credentials` → `API key`  
8. Скопируйте созданный ключ и сохраните его

# 🛡️ Как получить учетные данные OAuth 2.0?

- Приоритетное использование **OAuth 2.0**  
- Автоматическое переключение на метод **YouTube Data API Key**, если OAuth-данные отсутствуют  

1. Зайдите в [Google Cloud Console](https://console.cloud.google.com).
2. Выберите проект, для которого вы уже создали ключ YouTube Data API v3.
3. В левом столбце нажмите `OAuth consent screen`.
4. Выберите `Внешний` в качестве типа пользователя и нажмите `Создать`.
5. Заполните обязательные поля: **Название приложения**, **Электронная почта службы поддержки пользователей** и **Контактная информация разработчика** (для всех этих полей можно использовать свой адрес электронной почты). Нажмите `Сохранить и Продолжить`.
6. Вы можете пропустить страницу `Области действия`, нажав `Сохранить и Продолжить`.
7. На странице `Тестовые пользователи` нажмите `Добавить пользователей` и введите адрес электронной почты своей учетной записи Google. Нажмите `Сохранить и продолжить`.
8. Теперь снова нажмите `Учетные данные` в левом столбце.
9. В верхней части страницы нажмите `Создать учетные данные` → `Идентификатор клиента OAuth`.
10. Выберите `Настольное приложение` в качестве типа приложения. Название можно оставить по умолчанию. Нажмите `Создать`.
11. Появится окно. Нажмите `Скачать JSON`»`, чтобы сохранить учетные данные.
12. Загруженный `.json` файл поместите в папку с названием `Keys` в папке `bin`.

# Основные возможности

## 🔑 Обработка ключа YouTube API

- Первичная проверка введенного API-ключа перед запуском  
- Автоматическое создание и сохранение файла `Key.bin`  
- Отсутствие необходимости повторного ввода ключа после успешной проверки  

## 🚪 Проверка квоты

- Определение доступности текущей квоты YouTube API  
- Остановка выполнения, если дальнейшие запросы невозможны  

## 💌 Comment Explorer

- Получение списка комментариев в консоли  
- Фильтрация по ключевым словам  
- Сортировка `по времени` или `по релевантности`  
- Ограничение количества выводимых результатов  
- Определение канала, на котором размещено видео  
- Вывод причины ошибки при её возникновении  

## 📹 Video Explorer

- Поиск видео по запросу  
- Поиск с учетом региона  
- Фильтрация по `диапазону дат публикации` и `продолжительности видео`  
- Ограничение количества результатов  
- Для каждого видео выводится:  
   1. Название  
   2. Прямая ссылка  
   3. Количество просмотров  
   4. Количество лайков  
   5. Количество дизлайков  
   6. Количество комментариев  
   7. Дата публикации  
   8. Название канала и ссылка на канал  

## 📈 Channel Explorer

- Сбор основной статистики канала:  
  - количество подписчиков  
  - количество просмотров  
  - описание  
  - дата регистрации  
  - ID канала (UC...)  
  - Хэндл (@...)  
- Возможность поиска видео внутри канала

## 🗂️Playlist Explorer

- Ищите плейлисты по ключевым словам или безопасно загружайте **свои собственные** плейлисты с помощью `OAuth 2.0`
- Просматривайте основные сведения о плейлисте, такие как уровень конфиденциальности, общее количество видео и дата публикации
- Легко извлекайте все видео из любого плейлиста, введя его URL (поддерживаются ссылки в формате `PL...` и `OL...`)
- Получайте подробную статистику по каждому видео в плейлисте, в том числе количество дизлайков
- Анализируйте видео **одного плейлиста** сразу после вывода плейлистов, не возвращаясь в главное меню

## 📄 Subtitles Explorer

- Ввод двухбуквенного кода языка (`en`, `ru` и т.д.)  
- Выбор между `созданными вручную` и `автоматическими` субтитрами  
- Автоматическое предложение альтернативы при отсутствии выбранного типа  

## 🔢 Info Explorer

- Ввод URL видео для прямого получения метаданных  
- Возврат полного блока информации из Video Explorer + описание видео  

# ⚙️ Функциональность

- **Аутентификация** — приоритет отдается протоколу **OAuth 2.0** для безопасного доступа. Если учетные данные OAuth не найдены, приложение использует **ключ YouTube Data API**, хранящийся в файле `Key.bin`
- **Анализ ссылок** — `идентификатор видео`, `идентификатор канала` (UC...) и `хэштег` (@...) автоматически извлекаются из введенных вами ссылок
- **Фильтры поиска** — вы можете фильтровать видео по `региону`, `формату` (2D или 3D), `продолжительности` и `дате` (с помощью интеллектуального календаря)
- **Поиск по каналам** — в отличие от стандартного поиска, вы можете выполнять поиск по ключевым словам непосредственно в библиотеке канала
- **Интеллектуальные субтитры** — интегрирован **YouTube Transcript API**. Сначала он ищет субтитры, созданные `вручную`. Если их нет, приложение спрашивает, нужны ли вам субтитры, созданные `автоматически`
- **Дизлайки** — интегрирован **Return YouTube Dislike API**, который показывает количество лайков и дизлайков в сочетании с официальными данными
- **Фильтрация комментариев** — для локальной фильтрации комментариев и ответов используются **ключевые слова**. YouTube Data API не выполняет такую фильтрацию
- **Проверка квоты** — перед запуском выполняется тестовый запрос, чтобы убедиться, что ваша **квота YouTube API** не превышена
- **Автоматическая проверка** — перед отправкой API запросов YouTube Explorer проверяет соединение с `google.com`, чтобы убедиться, что оно активно. Если соединение прервалось, можно повторить попытку подключения
